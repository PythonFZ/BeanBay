Create 4 new model files following the EXACT existing codebase pattern (SQLAlchemy 1.x style with `Column`, `declarative_base`, UUID string PKs):

**1. `app/models/brew_method.py`** — BrewMethod model:
- `id`: String PK, `default=lambda: str(uuid.uuid4())`
- `name`: String, NOT NULL, UNIQUE (e.g., "Espresso", "V60", "AeroPress")
- `created_at`: DateTime, `server_default=func.now()`

**2. `app/models/equipment.py`** — Group 4 simple equipment models in one file:

- **Grinder**: id (String UUID PK), name (String NOT NULL), created_at (DateTime server_default)
- **Brewer**: id (String UUID PK), name (String NOT NULL), created_at (DateTime server_default)
- **Paper**: id (String UUID PK), name (String NOT NULL), created_at (DateTime server_default)
- **WaterRecipe**: id (String UUID PK), name (String NOT NULL), recipe_details (String nullable — freeform text for how it's made), created_at (DateTime server_default)

Use `__tablename__` values: `grinders`, `brewers`, `papers`, `water_recipes`

**3. `app/models/brew_setup.py`** — BrewSetup model:
- `id`: String UUID PK
- `name`: String, nullable (optional user label)
- `brew_method_id`: String, ForeignKey("brew_methods.id"), NOT NULL
- `grinder_id`: String, ForeignKey("grinders.id"), nullable
- `brewer_id`: String, ForeignKey("brewers.id"), nullable
- `paper_id`: String, ForeignKey("papers.id"), nullable
- `water_recipe_id`: String, ForeignKey("water_recipes.id"), nullable
- `created_at`: DateTime, server_default
- Relationships: `brew_method`, `grinder`, `brewer`, `paper`, `water_recipe` — all using `relationship("ClassName")`

**4. `app/models/bag.py`** — Bag model:
- `id`: String UUID PK
- `bean_id`: String, ForeignKey("beans.id"), NOT NULL, index=True
- `purchase_date`: Date, nullable
- `cost`: Float, nullable (META-03)
- `weight_grams`: Float, nullable
- `notes`: String, nullable
- `created_at`: DateTime, server_default
- Relationship: `bean = relationship("Bean", back_populates="bags")`

**5. Extend `app/models/bean.py`:**
- Add 3 new columns:
  - `roast_date = Column(Date, nullable=True)` — import Date from sqlalchemy
  - `process = Column(String, nullable=True)` — values: "washed", "natural", "honey", "anaerobic", "other"
  - `variety = Column(String, nullable=True)` — freeform text
- Add relationship: `bags = relationship("Bag", back_populates="bean", cascade="all, delete-orphan")`

**6. Extend `app/models/measurement.py`:**
- Add column: `brew_setup_id = Column(String, ForeignKey("brew_setups.id"), nullable=True, index=True)`
- Add relationship: `brew_setup = relationship("BrewSetup")`
- Import ForeignKey is already present; just add the column and relationship

**7. Update `app/models/__init__.py`:**
- Import all new models: BrewMethod, Grinder, Brewer, Paper, WaterRecipe, BrewSetup, Bag
- Add all to `__all__`
- This is CRITICAL: `migrations/env.py` imports `app.models` to detect models for autogenerate, and `tests/conftest.py` imports from here to register models with Base

**IMPORTANT conventions to follow:**
- Use `from app.database import Base` (not declarative_base())
- Use `Column()` not `mapped_column()` (match existing style)
- Use `back_populates` not `backref`
- Use string references in relationships: `relationship("ClassName")` to avoid circular imports
- All new entity models use String UUID PKs (not Integer)
