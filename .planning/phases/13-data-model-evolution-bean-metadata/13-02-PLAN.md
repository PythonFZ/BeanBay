**Step 3: Add data migration to the upgrade function**

After ALL table creations and column additions, add data migration using `op.execute()`:

```python
# Seed default brew method and setup for existing data
DEFAULT_METHOD_ID = "00000000-0000-0000-0000-000000000001"
DEFAULT_SETUP_ID = "00000000-0000-0000-0000-000000000002"

op.execute(
    f"INSERT INTO brew_methods (id, name, created_at) "
    f"VALUES ('{DEFAULT_METHOD_ID}', 'Espresso', CURRENT_TIMESTAMP)"
)
op.execute(
    f"INSERT INTO brew_setups (id, brew_method_id, created_at) "
    f"VALUES ('{DEFAULT_SETUP_ID}', '{DEFAULT_METHOD_ID}', CURRENT_TIMESTAMP)"
)
op.execute(
    f"UPDATE measurements SET brew_setup_id = '{DEFAULT_SETUP_ID}'"
)
```

**CRITICAL: Use hardcoded, deterministic UUIDs** (`00000000-0000-0000-0000-000000000001` and `00000000-0000-0000-0000-000000000002`) so the migration is idempotent and the downgrade can reference the same IDs.

**Step 4: Add proper downgrade**

The downgrade must:
1. Remove brew_setup_id from measurements (via batch_alter_table)
2. Remove roast_date, process, variety from beans (via batch_alter_table)
3. Drop tables in reverse dependency order: bags, brew_setups, water_recipes, papers, brewers, grinders, brew_methods
4. The seeded data rows will be automatically deleted when the tables are dropped

**IMPORTANT:**
- Order of `create_table` matters: brew_methods MUST be created before brew_setups (FK dependency)
- Order of `drop_table` in downgrade: brew_setups before brew_methods, bags before beans columns
- Use `batch_alter_table` for ALL column operations on existing tables (SQLite requirement)
- The `server_default=sa.text('(CURRENT_TIMESTAMP)')` pattern is used in existing migrations
- Do NOT create indexes separately for columns that already specify `index=True` in the model — Alembic autogenerate handles this
