  <action>
Create `app/routers/analytics.py` with a single GET `/analytics` route that:

1. Queries ALL measurements (across all beans) to compute aggregate stats:
   - `total_shots`: count of all measurements
   - `total_beans`: count of distinct beans with at least 1 measurement
   - `avg_taste`: mean taste score (exclude failed shots, round to 1 decimal)
   - `best_taste`: highest taste score ever (exclude failed)
   - `best_bean_name`: name of the bean that produced the best taste
   - `total_failed`: count of failed shots
   - `improvement_rate`: compare average taste of last 10 non-failed shots vs first 10 non-failed shots (show "↑ X.X" or "↓ X.X" or "—" if < 10 shots total)

2. Queries best recipe per bean for comparison:
   - For each bean that has at least 1 non-failed measurement, find the measurement with the highest taste score
   - Return a list of dicts: `{bean_name, taste, grind_setting, temperature, preinfusion_pct, dose_in, target_yield, saturation, shot_count}`
   - Sort by taste descending (best bean first)

3. Passes `stats`, `comparison`, and `active_bean` to template.

Use `_get_active_bean` from beans router (import like history.py does). Use `TemplateResponse(request, "analytics/index.html", {...})` signature (new style, no deprecation warnings).

Create `app/templates/analytics/index.html` extending base.html with:
- Page title "Analytics"
- Include `_stats_card.html` partial for stats
- Include `_comparison_table.html` partial for comparison
- Empty state if total_shots == 0: "Pull some shots to see analytics."

Create `app/templates/analytics/_stats_card.html`:
- A `.card` div showing stats in a 2-column grid:
  - Total shots / Total beans (top row)
  - Avg taste / Best taste (second row, best taste highlighted with bean name)
  - Failed shots / Improvement rate (third row)
- Use existing CSS variables and card patterns from insights page
- Numbers should be large (`var(--text-2xl)`) and scannable

Create `app/templates/analytics/_comparison_table.html`:
- A `.card` with title "Best Recipes by Bean"
- If no comparison data: "No beans with shots yet."
- Otherwise: a vertically-stacked list of bean cards (NOT a horizontal table — too wide for mobile)
- Each bean card shows:
  - Bean name + taste score (header)
  - 2x3 grid of recipe params: grind, temp, preinfusion%, dose, yield, saturation
  - Shot count badge ("12 shots")
- Use `.card` nesting pattern. Each param displayed as label (small, muted) above value (large).

Register the router in `app/main.py`:
- `from app.routers import ... analytics` (add to existing import)
- `app.include_router(analytics.router)`

Add "Analytics" link to navigation in `app/templates/base.html`:
- Add `<a href="/analytics" class="nav-link">Analytics</a>` after the Insights link

Add CSS to `app/static/css/main.css` for analytics-specific styles:
- `.stats-grid` — 2-column grid, gap: var(--spacing-md)
- `.stat-item` — flex column, center-aligned
- `.stat-value` — font-size: var(--text-2xl), font-weight bold, color: var(--text-primary)
- `.stat-label` — font-size: var(--text-sm), color: var(--text-muted)
- `.stat-highlight` — color: var(--accent) (for best taste)
- `.comparison-bean` — card-like container for each bean's best recipe
- `.recipe-grid` — 3-column grid for recipe params on mobile (2x3 layout)
- `.recipe-param-label` — small muted text
- `.recipe-param-value` — larger text, monospace for numbers
