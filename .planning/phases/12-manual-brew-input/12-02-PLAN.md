**Route (brew.py):**

1. Add `GET /brew/manual` route to `brew.py`. It must:
   - Require active bean (redirect to /beans if none)
   - Load the best measurement for the bean (for pre-fill values)
   - Resolve parameter bounds using `_resolve_bounds(bean.parameter_overrides)` — import from `app.services.optimizer`
   - Generate a UUID for `recommendation_id` (same pattern as best.html: `manual_session_id = str(uuid.uuid4())`)
   - Compute pre-fill values:
     - If best measurement exists: use its grind_setting, temperature, preinfusion_pct, dose_in, target_yield, saturation
     - Otherwise: use midpoint of each bound range. For saturation, default to "yes"
   - Pass to template: `active_bean`, `bounds` (dict of param -> (min, max)), `defaults` (dict of param -> value), `default_saturation` ("yes"/"no"), `manual_session_id`

**Template (manual.html):**

2. Create `app/templates/brew/manual.html` extending `base.html`. Structure:
   - Page header: "Manual Brew" with "← Back" link to /brew
   - Card with bean name subtitle
   - Form with `method="post" action="/brew/record"`:
     - Hidden inputs: `recommendation_id` (= manual_session_id), `is_manual` (= "true")
     - **5 continuous parameters** — for each param (grind_setting, temperature, preinfusion_pct, dose_in, target_yield):
       - Label (e.g. "Grind Setting")
       - Container div with class `param-input-row` containing:
         - Range slider: `min="{{ bounds[param][0] }}" max="{{ bounds[param][1] }}" step="<step>" value="{{ defaults[param] }}"` with class `param-slider`
         - Number input: same min/max/step/value, class `param-number`, name is the param name (e.g. `name="grind_setting"`)
       - Bidirectional sync: slider `oninput` updates number input value; number input `oninput` updates slider value. Use inline JS:
         ```
         oninput="document.getElementById('num-{param}').value = this.value"
         ```

         and vice versa for the number input.
       - Steps per param: grind_setting=0.5, temperature=1, preinfusion_pct=5, dose_in=0.5, target_yield=1
       - Units shown after the number input: °C for temperature, % for preinfusion_pct, g for dose_in and target_yield, none for grind_setting
     - **Saturation toggle:**
       - Hidden input with `name="saturation" value="no"`
       - Checkbox input with `name="saturation" value="yes"`, checked if default_saturation is "yes"
       - Label: "Saturation"

       - Required message div with id `taste-required-msg`

**Extract toggleFailed (eliminate 3x duplication):**

1. Move the `toggleFailed` function from inline `<script>` blocks into `app/static/js/tags.js`. Add it inside the IIFE, and expose as `window.toggleFailed`:
   ```javascript
   window.toggleFailed = function(checkbox) {
     var tasteGroup = document.getElementById('taste-group');
     var tasteInput = document.getElementById('taste');
     var tasteDisplay = document.getElementById('taste-display');
     var requiredMsg = document.getElementById('taste-required-msg');
     if (checkbox.checked) {
       tasteInput.value = '1';
       tasteInput.dataset.touched = 'true';
       tasteDisplay.textContent = '1.0';
       tasteGroup.classList.add('touched');
       tasteGroup.style.opacity = '0.4';
       tasteGroup.style.pointerEvents = 'none';
       if (requiredMsg) requiredMsg.classList.remove('visible');
     } else {
       tasteInput.value = '7.0';
       tasteInput.dataset.touched = 'false';
       tasteDisplay.textContent = '\u2014';
       tasteGroup.classList.remove('touched');
       tasteGroup.style.opacity = '';
       tasteGroup.style.pointerEvents = 'auto';
     }
   };
   ```

2. Remove the inline `<script>` blocks containing `toggleFailed` from `recommend.html` and `best.html`. Keep only `<script src="/static/js/tags.js"></script>` in their `{% block scripts %}`.
